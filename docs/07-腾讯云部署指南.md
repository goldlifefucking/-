# 校园工单管理系统 - 腾讯云服务器部署指南

## 服务器信息
- 系统: Ubuntu
- 配置: 2核4G
- 部署方式: Docker Compose (PostgreSQL + NestJS API + Nginx)

---

## 第一阶段：连接服务器

### 1.1 获取服务器信息
在腾讯云控制台找到：
- **公网 IP**: 例如 `43.136.xxx.xxx`
- **初始密码**: 首次登录需要重置密码或使用密钥

### 1.2 SSH 连接服务器
在你电脑的 cmd 或 PowerShell 中：
```bash
ssh root@你的公网IP
```
输入密码后登录。

> 如果使用密钥登录，参考腾讯云文档配置密钥对。

---

## 第二阶段：安装 Docker 环境

### 2.1 更新系统
```bash
apt update && apt upgrade -y
```

### 2.2 安装 Docker
```bash
# 安装 Docker
curl -fsSL https://get.docker.com | bash

# 验证安装
docker --version
docker compose version
```

### 2.3 启动 Docker 服务
```bash
systemctl start docker
systemctl enable docker
```

## 第三阶段：上传项目代码

### 方法 A：从 GitHub 克隆（推荐，服务器在国内可直连 Gitee）

#### 3A.1 在 Gitee 创建镜像仓库
1. 打开 https://gitee.com 登录
2. 右上角 "+" → "从 GitHub/GitLab 导入仓库"
3. 输入 GitHub 仓库地址: `https://github.com/goldlifefucking/-`
4. 导入完成后，Gitee 会给你一个地址，类似: `https://gitee.com/你的用户名/-`

#### 3A.2 在服务器上克隆
```bash
cd /opt
git clone https://gitee.com/你的用户名/-  campus-workorder
cd campus-workorder
```

### 方法 B：直接从 GitHub 克隆（如果服务器能访问 GitHub）
```bash
cd /opt
git clone https://github.com/goldlifefucking/-  campus-workorder
cd campus-workorder
```

### 方法 C：用 SCP 上传（如果以上都不行）
在你电脑的 cmd 中：
```cmd
# 先把项目打包（排除不需要的文件）
cd "D:\校园工单管理系统"
tar -czf project.tar.gz --exclude=node_modules --exclude=.git --exclude=.codebuddy --exclude=.workbuddy --exclude=*.pptx .

# 上传到服务器
scp project.tar.gz root@你的公网IP:/opt/

# SSH 登录服务器后解压
ssh root@你的公网IP
cd /opt
mkdir -p campus-workorder
cd campus-workorder
tar -xzf ../project.tar.gz
```

## 第四阶段：配置环境变量

### 4.1 创建 .env 文件
在服务器项目目录中：
```bash
cd /opt/campus-workorder
cp .env.example .env
```

### 4.2 编辑 .env 文件
```bash
nano .env
```

修改为以下内容（**把密码改成你自己的**）：
```
# PostgreSQL
DB_PASSWORD=你的强密码_比如_Campus2026!Secure

# JWT
JWT_SECRET=你的随机密钥_比如_a8f3k2j9s5d7f1g4h6
JWT_EXPIRES_IN=7d

# 前端
VITE_API_BASE_URL=/api
VITE_SOCKET_URL=/api/socket
```

保存退出：`Ctrl+O` 回车保存，`Ctrl+X` 退出

### 4.3 同步修改 docker-compose 中的密码
```bash
nano docker-compose.prod.yml
```

确认 `DB_PASSWORD` 和 `DATABASE_URL` 中的密码与 .env 文件一致。

## 第五阶段：构建和启动

### 5.1 构建并启动所有服务
```bash
cd /opt/campus-workorder
docker compose -f docker-compose.prod.yml up -d --build
```

> 构建过程大约需要 5-10 分钟（下载镜像 + 安装依赖 + 编译）
> 等待出现 "Started" 或 "Running" 字样

### 5.2 查看启动状态
```bash
# 查看容器状态
docker compose -f docker-compose.prod.yml ps

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 单独看 API 日志（确认数据库迁移和种子数据）
docker compose -f docker-compose.prod.yml logs api
```

### 5.3 确认 API 启动成功
看到以下日志说明成功：
```
>>> 数据库表结构已同步
>>> 初始化种子数据...
[Nest] LOG [Bootstrap] 应用已启动: http://localhost:3000
[Nest] LOG [Bootstrap] Swagger 文档: http://localhost:3000/api/docs
```

## 第六阶段：配置腾讯云安全组

### 6.1 开放端口
在腾讯云控制台：
1. 进入 **轻量应用服务器** 或 **云服务器 CVM**
2. 点击你的实例 → **防火墙** 或 **安全组**
3. 添加规则：

| 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|
| TCP | 80 | 0.0.0.0/0 | HTTP 访问 |
| TCP | 443 | 0.0.0.0/0 | HTTPS（后续配域名时用） |
| TCP | 22 | 你的IP/32 | SSH（建议限制来源） |

> **重要**: 不要开放 3000 和 5432 端口，它们只在 Docker 内部网络通信

### 6.2 验证访问
打开浏览器访问：
```
http://你的公网IP/
```

应该看到登录页面。

### 6.3 测试 API
```
http://你的公网IP/api/health
```
应返回 `{"status":"ok"}`

### 6.4 测试登录
用默认账号登录：
- 管理员: `admin` / `Admin123456`
- 学生: `student` / `Student123456`


## 常见问题

### Q: 访问 http://IP 显示 502 Bad Gateway
```bash
# 检查 API 是否正常
docker compose -f docker-compose.prod.yml logs api
# 如果 API 没起来，可能是数据库连接失败，检查 .env 配置
```

### Q: API 日志显示 "数据库连接超时"
```bash
# 检查 PostgreSQL 是否正常
docker compose -f docker-compose.prod.yml logs postgres
# 检查 .env 中的 DB_PASSWORD 是否与 docker-compose.prod.yml 一致
```

### Q: 构建时 pnpm install 失败
```bash
# 可能是网络问题，配置 npm 镜像
docker compose -f docker-compose.prod.yml build --no-cache api
```

### Q: 磁盘空间不足
```bash
# 清理 Docker 无用镜像
docker system prune -a -f
```

### Q: 想要配置 HTTPS（有域名后）
```bash
# 安装 certbot
apt install -y certbot

# 在 docker-compose.prod.yml 中把 web 的 80 端口改为 8080
# 然后用 certbot 获取证书
certbot certonly --standalone -d 你的域名

# 修改 nginx.conf 添加 SSL 配置
```

## 架构说明

```
浏览器 → http://公网IP:80 → Nginx (web容器)
                              ├── / → 前端静态文件 (Vue3)
                              ├── /api/ → 反代到 API (api容器:3000)
                              ├── /api/socket/ → WebSocket 反代
                              └── /uploads/ → 文件服务反代

API (api容器:3000) → PostgreSQL (postgres容器:5432)
```

所有容器在同一个 Docker 网络中，端口 3000 和 5432 不对外暴露，只有 Nginx 的 80 端口对外。
