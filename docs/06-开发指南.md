## 开发流程

### 1. 启动开发环境

```bash
# 启动 PostgreSQL
docker compose up -d postgres

# 安装依赖
pnpm install

# 初始化数据库
pnpm prisma:generate
pnpm prisma:push
pnpm prisma:seed

# 启动开发服务器
pnpm dev
```

### 2. 后端开发

每个业务模块遵循 NestJS 的模块化结构：

```
module-name/
├── module-name.controller.ts   # HTTP 接口
├── module-name.service.ts      # 业务逻辑
├── module-name.module.ts       # 模块定义
└── dto.ts                      # 数据传输对象
```

**开发新接口的步骤：**

1. 在 `dto.ts` 中定义请求参数 DTO，使用 `class-validator` 装饰器
2. 在 `service.ts` 中实现业务逻辑
3. 在 `controller.ts` 中定义 HTTP 端点，添加 Swagger 装饰器
4. 在 `module.ts` 中注册 Controller 和 Service

**权限控制：**

```typescript
@UseGuards(JwtAuthGuard, RolesGuard)  // 登录验证 + 角色验证
@Roles(UserRole.ADMIN)                 // 仅管理员
@Get()
findAll(@CurrentUser() user: any) {
  // user 包含 id, username, name, role
}
```

### 3. 前端开发

**API 调用：**

```typescript
// src/api/ 目录下定义 API
import http from './http';
export const exampleApi = {
  findAll: () => http.get('/example'),
  create: (data: any) => http.post('/example', data),
};
```

**状态管理（Pinia）：**

```typescript
// src/stores/ 目录下定义 Store
export const useExampleStore = defineStore('example', () => {
  const data = ref([]);
  const fetchData = async () => { data.value = await exampleApi.findAll(); };
  return { data, fetchData };
});
```

**路由：**

路由按角色分组，使用路由守卫进行权限控制。新增页面时在 `router/index.ts` 中添加路由配置。

### 4. 数据库变更

修改 `prisma/schema.prisma` 后：

```bash
pnpm prisma:generate  # 重新生成 Prisma Client
pnpm prisma:push      # 推送变更到数据库
```

### 5. 实时通信

**后端推送：**

```typescript
// 在 Service 中注入 RealtimeGateway
constructor(private realtimeGateway: RealtimeGateway) {}

// 推送工单更新
this.realtimeGateway.emitWorkOrderUpdated(userId, workOrder);
// 推送通知
this.realtimeGateway.emitNotificationCreated(userId, notification);
```

**前端监听：**

Socket.IO 连接在 Layout 组件的 `onMounted` 中建立。事件监听在 `stores/socket.ts` 中定义，收到事件后更新对应的 Pinia Store，UI 自动响应式更新。

## 代码规范

### TypeScript

- 严格模式：启用 `strict: true`
- 类型注解：公共 API 必须有明确类型
- 枚举：使用 `@campus-work-order/shared` 中定义的共享枚举

### 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| 文件名 | kebab-case | `work-orders.service.ts` |
| 类名 | PascalCase | `WorkOrdersService` |
| 方法名 | camelCase | `findAll()` |
| 常量 | UPPER_SNAKE_CASE | `STATUS_TRANSITIONS` |
| Vue 组件 | PascalCase | `WorkOrderStatusTag.vue` |

## 调试技巧

### 后端调试

1. **Swagger**：访问 `http://localhost:3000/api/docs`，使用 Bearer Token 测试接口
2. **Prisma Studio**：运行 `pnpm prisma:studio` 可视化查看数据库
3. **日志**：NestJS 默认输出控制台日志

### 前端调试

1. **Vue DevTools**：安装浏览器插件查看组件状态
2. **Pinia DevTools**：查看状态变化
3. **Network**：浏览器开发者工具查看 API 请求
4. **Socket.IO**：控制台查看 WebSocket 连接和事件

## 常见问题

### Q: 修改了 Prisma Schema 后前端类型不更新？

A: 运行 `pnpm prisma:generate` 重新生成 Prisma Client，然后重启后端。

### Q: Socket.IO 连接不上？

A: 检查后端是否启动、Nginx/Vite 代理是否正确配置 WebSocket 升级。

### Q: 图片上传后无法显示？

A: 检查后端静态文件服务配置（`app.useStaticAssets`）和 Nginx/Vite 代理配置。

### Q: 角色权限不生效？

A: 检查 Controller 上的 `@UseGuards(JwtAuthGuard, RolesGuard)` 和 `@Roles()` 装饰器是否正确配置。
