## 3. 技术栈选型理由

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端框架 | Vue 3 (Composition API) | 响应式编程、组件化开发、生态成熟 |
| 构建工具 | Vite | 极速冷启动、HMR 热更新 |
| UI 组件库 | Ant Design Vue 4 | 企业级组件库，表格/表单/图表支持好 |
| 状态管理 | Pinia | Vue 3 官方推荐，TypeScript 友好 |
| HTTP 客户端 | Axios | 拦截器支持、请求/响应统一处理 |
| 图表 | ECharts | 数据大屏可视化首选 |
| 实时通信 | Socket.IO | 支持 WebSocket 降级，事件驱动 |
| 后端框架 | NestJS | 模块化架构，依赖注入，TypeScript 原生 |
| ORM | Prisma | 类型安全、迁移管理、数据建模直观 |
| 数据库 | PostgreSQL | 关系型数据库，支持 JSON，性能优秀 |
| 认证 | JWT + bcrypt | 无状态认证 + 安全密码存储 |
| 文档 | Swagger | NestJS 原生集成，自动生成 API 文档 |
| 容器化 | Docker + Docker Compose | 环境一致性，一键部署 |
| Web 服务器 | Nginx | 反向代理、静态文件服务、高性能 |
| 包管理 | pnpm | 磁盘空间高效，严格依赖管理，monorepo 原生支持 |

## 4. 后端模块划分

### 4.1 模块依赖关系

```
app.module
├── auth.module ─────── 依赖 users.module, prisma.module
├── users.module ────── 依赖 prisma.module
├── categories.module ─ 依赖 prisma.module
├── work-orders.module ─ 依赖 prisma.module, notifications.module, realtime.module
├── notifications.module 依赖 prisma.module
├── files.module ─────── 依赖 prisma.module
├── realtime.module ──── 独立模块 (Socket.IO Gateway)
├── common/ ──────────── 被所有模块引用 (Guards/Decorators/Filters/Pipes)
└── prisma.module ───── 全局模块 (@Global)
```

### 4.2 各模块职责

| 模块 | 职责 |
|------|------|
| `auth` | 用户登录、JWT Token 签发与验证 |
| `users` | 用户 CRUD、密码加密、维修人员查询 |
| `categories` | 报修分类的增删改查 |
| `work-orders` | 工单 CRUD + 状态流转（受理/派单/处理/确认/取消） |
| `notifications` | 通知的创建、列表查询、已读标记、未读计数 |
| `files` | 文件上传到本地或 OSS |
| `realtime` | Socket.IO Gateway，工单状态变更时推送实时通知 |
| `common` | 公共装饰器 (`@Roles`, `@CurrentUser`)、守卫 (`JwtAuthGuard`, `RolesGuard`)、过滤器、拦截器、管道 |
| `prisma` | PrismaService (全局模块)，封装 PrismaClient |

## 5. 前端结构说明

### 5.1 目录职责

| 目录 | 职责 |
|------|------|
| `api/` | Axios 封装 + 按模块拆分的请求函数，统一错误处理 |
| `router/` | Vue Router 配置，路由守卫（未登录跳转） |
| `stores/` | Pinia Store，管理全局状态（用户/工单/通知/Socket） |
| `layouts/` | 默认布局（侧边栏+顶栏+内容区），空白布局（登录页） |
| `components/` | 可复用组件（状态标签、图片上传、时间线等） |
| `views/` | 页面组件，每个路由对应一个 View |

### 5.2 路由设计

```typescript
// 路由配置
const routes = [
  { path: '/login',             component: Login,          meta: { public: true } },
  { path: '/',                  component: Dashboard,      meta: { title: '工作台' } },
  { path: '/work-orders',       component: WorkOrderList,  meta: { title: '工单列表' } },
  { path: '/work-orders/create',component: WorkOrderCreate, meta: { title: '创建工单', roles: ['STUDENT','TEACHER'] } },
  { path: '/work-orders/:id',   component: WorkOrderDetail,meta: { title: '工单详情' } },
  { path: '/users',             component: UserList,       meta: { title: '用户管理', roles: ['ADMIN'] } },
  { path: '/categories',        component: CategoryList,   meta: { title: '分类管理', roles: ['ADMIN'] } },
  { path: '/statistics',        component: Statistics,     meta: { title: '数据统计', roles: ['ADMIN'] } },
  { path: '/notifications',     component: NotificationList,meta: { title: '通知中心' } },
];
```

### 5.3 状态管理

| Store | 管理内容 |
|-------|---------|
| `useAuthStore` | 当前用户信息、Token、登录/登出方法 |
| `useWorkOrderStore` | 工单列表、当前工单、筛选条件 |
| `useNotificationStore` | 通知列表、未读计数 |
| `useSocketStore` | Socket.IO 连接实例、连接状态、事件监听 |

## 6. 实时通信架构

### 6.1 通信流程

```
┌─────────────────┐          WebSocket          ┌─────────────────┐
│   Vue3 前端      │◄───────────────────────────►│   NestJS 后端    │
│                  │    socket.io-client         │                  │
│  useSocketStore  │◄────── 事件 ───────────────►│  RealtimeGateway │
│   ├─ 监听通知     │    notification:new         │   ├─ 全局连接管理  │
│   ├─ 监听工单更新  │    work-order:statusChanged  │   ├─ 命名空间 /   │
│   └─ 自动更新UI   │                             │   └─ 事件广播     │
└─────────────────┘                             └─────────────────┘
```

### 6.2 事件定义

| 事件名 | 方向 | 触发条件 | 携带数据 |
|--------|------|----------|----------|
| `notification:new` | 服务端→客户端 | 新通知创建 | `{ id, title, content, type, workOrderId }` |
| `work-order:statusChanged` | 服务端→客户端 | 工单状态变更 | `{ workOrderId, status, timestamp }` |
| `connection:authenticate` | 客户端→服务端 | WebSocket 连接成功后 | `{ token }` |

### 6.3 推送策略

- 工单状态变更时，向关联用户（报修人/维修人员/管理员）所在房间广播
- 用户连接时自动加入 `user:{userId}` 房间
- 管理员加入 `role:ADMIN` 房间，接收全局通知

### 7.2 生产环境配置文件示例

**docker-compose.prod.yml:**

```yaml
version: '3.8'
services:
  postgres:
    image: postgres:16-alpine
    restart: always
    environment:
      POSTGRES_DB: campus_work_order
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    networks:
      - app-net

  api:
    build:
      context: .
      dockerfile: apps/api/Dockerfile
    restart: always
    depends_on:
      - postgres
    environment:
      DATABASE_URL: postgresql://postgres:${DB_PASSWORD}@postgres:5432/campus_work_order
      JWT_SECRET: ${JWT_SECRET}
      UPLOAD_DIR: /app/uploads
    volumes:
      - uploads:/app/uploads
    networks:
      - app-net

  web:
    build:
      context: .
      dockerfile: apps/web/Dockerfile
    restart: always
    ports:
      - "80:80"
    depends_on:
      - api
    networks:
      - app-net

volumes:
  pgdata:
  uploads:

networks:
  app-net:
    driver: bridge
```

### 7.3 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name localhost;

    # 文件上传大小限制
    client_max_body_size 10m;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://api:3000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Swagger 文档
    location /api/docs/ {
        proxy_pass http://api:3000/api/docs/;
        proxy_set_header Host $host;
    }

    # WebSocket (Socket.IO)
    location /socket.io/ {
        proxy_pass http://api:3000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```
