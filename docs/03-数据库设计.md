# 03 - 数据库设计

## 1. ER 关系说明

### 1.1 核心关系

```
┌──────────┐          ┌──────────────┐          ┌──────────┐
│  users   │◄────────┤  work_orders  │─────────►│categories│
│          │ reporterId│              │ categoryId│          │
└──────────┘  workerId │              │          └──────────┘
                       │              │
                       └──────┬───────┘
                              │
              ┌───────────────┼───────────────┐
              │               │               │
         ┌────▼────┐   ┌─────▼─────┐   ┌─────▼──────┐
         │attachments│  │   logs    │   │notifications│
         └──────────┘   └───────────┘   └────────────┘
```

### 1.2 表关系总结

| 关系 | 说明 |
|------|------|
| `users` 1:N `work_orders` (reporterId) | 一个用户可提交多个工单 |
| `users` 1:N `work_orders` (workerId) | 一个维修人员可处理多个工单 |
| `categories` 1:N `work_orders` (categoryId) | 一个分类下有多个工单 |
| `work_orders` 1:N `attachments` | 一个工单可有多张附件图片 |
| `work_orders` 1:N `logs` | 一个工单可有多条操作日志 |
| `work_orders` 1:N `notifications` | 一个工单可产生多条通知 |

## 3. 表结构详细设计

### 3.1 users (用户表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 用户ID |
| `username` | `String` | UNIQUE, NOT NULL | 用户名（登录用） |
| `password` | `String` | NOT NULL | 密码（bcrypt 加密） |
| `name` | `String` | NOT NULL | 真实姓名 |
| `role` | `UserRole` | NOT NULL | 用户角色 |
| `phone` | `String?` | | 联系电话 |
| `avatar` | `String?` | | 头像URL |
| `isActive` | `Boolean` | NOT NULL, DEFAULT true | 账号是否启用 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 创建时间 |
| `updatedAt` | `DateTime` | NOT NULL, @updatedAt | 更新时间 |

**Prisma 模型:**

```prisma
model User {
  id           Int          @id @default(autoincrement())
  username     String       @unique
  password     String
  name         String
  role         UserRole
  phone        String?
  avatar       String?
  isActive     Boolean      @default(true)
  createdAt    DateTime     @default(now())
  updatedAt    DateTime     @updatedAt
  // 关联
  reportedOrders   WorkOrder[] @relation("Reporter")
  assignedOrders   WorkOrder[] @relation("Worker")
  logs             Log[]
  notifications    Notification[]
}
```

### 3.2 categories (分类表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 分类ID |
| `name` | `String` | UNIQUE, NOT NULL | 分类名称 |
| `description` | `String?` | | 分类描述 |
| `icon` | `String?` | | 分类图标（Ant Design 图标名） |
| `sortOrder` | `Int` | NOT NULL, DEFAULT 0 | 排序序号 |
| `isActive` | `Boolean` | NOT NULL, DEFAULT true | 是否启用 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 创建时间 |
| `updatedAt` | `DateTime` | NOT NULL, @updatedAt | 更新时间 |

**Prisma 模型:**

```prisma
model Category {
  id          Int         @id @default(autoincrement())
  name        String      @unique
  description String?
  icon        String?
  sortOrder   Int         @default(0)
  isActive    Boolean     @default(true)
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
  // 关联
  workOrders  WorkOrder[]
}
```

### 3.3 work_orders (工单表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 工单ID |
| `title` | `String` | NOT NULL | 工单标题 |
| `description` | `String` | NOT NULL | 问题描述 |
| `location` | `String` | NOT NULL | 报修位置 |
| `priority` | `Priority` | NOT NULL, DEFAULT MEDIUM | 优先级 |
| `status` | `WorkOrderStatus` | NOT NULL, DEFAULT PENDING | 当前状态 |
| `resolution` | `String?` | | 维修处理结果 |
| `feedback` | `String?` | | 报修人反馈意见 |
| `categoryId` | `Int` | FK → categories.id, NOT NULL | 报修分类ID |
| `reporterId` | `Int` | FK → users.id, NOT NULL | 报修人ID |
| `workerId` | `Int?` | FK → users.id | 指派的维修人员ID |
| `acceptedAt` | `DateTime?` | | 受理时间 |
| `assignedAt` | `DateTime?` | | 派单时间 |
| `startedAt` | `DateTime?` | | 开始处理时间 |
| `resolvedAt` | `DateTime?` | | 提交结果时间 |
| `confirmedAt` | `DateTime?` | | 确认完成时间 |
| `cancelledAt` | `DateTime?` | | 取消时间 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 创建时间 |
| `updatedAt` | `DateTime` | NOT NULL, @updatedAt | 更新时间 |

**Prisma 模型:**

```prisma
model WorkOrder {
  id              Int              @id @default(autoincrement())
  title           String
  description     String
  location        String
  priority        Priority         @default(MEDIUM)
  status          WorkOrderStatus  @default(PENDING)
  resolution      String?
  feedback        String?
  categoryId      Int
  reporterId      Int
  workerId        Int?
  acceptedAt      DateTime?
  assignedAt      DateTime?
  startedAt       DateTime?
  resolvedAt      DateTime?
  confirmedAt     DateTime?
  cancelledAt     DateTime?
  createdAt       DateTime         @default(now())
  updatedAt       DateTime         @updatedAt
  // 关联
  category        Category         @relation(fields: [categoryId], references: [id])
  reporter        User             @relation("Reporter", fields: [reporterId], references: [id])
  worker          User?            @relation("Worker", fields: [workerId], references: [id])
  attachments     Attachment[]
  logs            Log[]
  notifications   Notification[]
  // 索引
  @@index([status])
  @@index([reporterId])
  @@index([workerId])
  @@index([categoryId])
}
```

### 3.4 attachments (附件表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 附件ID |
| `workOrderId` | `Int` | FK → work_orders.id, NOT NULL | 所属工单ID |
| `url` | `String` | NOT NULL | 文件访问URL |
| `filename` | `String` | NOT NULL | 原始文件名 |
| `mimeType` | `String` | NOT NULL | MIME类型 |
| `size` | `Int` | NOT NULL | 文件大小（字节） |
| `type` | `AttachmentType` | NOT NULL | 附件类型 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 上传时间 |

**Prisma 模型:**

```prisma
model Attachment {
  id          Int            @id @default(autoincrement())
  workOrderId Int
  url         String
  filename    String
  mimeType    String
  size        Int
  type        AttachmentType
  createdAt   DateTime       @default(now())
  // 关联
  workOrder   WorkOrder      @relation(fields: [workOrderId], references: [id], onDelete: Cascade)
  // 索引
  @@index([workOrderId])
}
```

### 3.5 logs (操作日志表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 日志ID |
| `workOrderId` | `Int` | FK → work_orders.id, NOT NULL | 所属工单ID |
| `userId` | `Int` | FK → users.id, NOT NULL | 操作人ID |
| `action` | `String` | NOT NULL | 操作动作 |
| `fromStatus` | `String?` | | 变更前状态 |
| `toStatus` | `String?` | | 变更后状态 |
| `detail` | `String?` | | 操作详情 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 操作时间 |

**Prisma 模型:**

```prisma
model Log {
  id          Int       @id @default(autoincrement())
  workOrderId Int
  userId      Int
  action      String
  fromStatus  String?
  toStatus    String?
  detail      String?
  createdAt   DateTime  @default(now())
  // 关联
  workOrder   WorkOrder @relation(fields: [workOrderId], references: [id], onDelete: Cascade)
  user        User      @relation(fields: [userId], references: [id])
  // 索引
  @@index([workOrderId])
  @@index([userId])
}
```

### 3.6 notifications (通知表)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `Int` | PK, AUTO_INCREMENT | 通知ID |
| `userId` | `Int` | FK → users.id, NOT NULL | 接收通知的用户ID |
| `workOrderId` | `Int?` | FK → work_orders.id | 关联工单ID |
| `title` | `String` | NOT NULL | 通知标题 |
| `content` | `String` | NOT NULL | 通知内容 |
| `type` | `NotificationType` | NOT NULL | 通知类型 |
| `isRead` | `Boolean` | NOT NULL, DEFAULT false | 是否已读 |
| `createdAt` | `DateTime` | NOT NULL, DEFAULT now() | 通知时间 |

**Prisma 模型:**

```prisma
model Notification {
  id          Int              @id @default(autoincrement())
  userId      Int
  workOrderId Int?
  title       String
  content     String
  type        NotificationType
  isRead      Boolean          @default(false)
  createdAt   DateTime         @default(now())
  // 关联
  user        User             @relation(fields: [userId], references: [id])
  workOrder   WorkOrder?       @relation(fields: [workOrderId], references: [id], onDelete: Cascade)
  // 索引
  @@index([userId, isRead])
  @@index([userId, createdAt])
}
```

## 4. 索引设计说明

| 表 | 索引字段 | 索引类型 | 使用场景 |
|----|----------|----------|----------|
| `users` | `username` | UNIQUE | 登录查询 |
| `work_orders` | `status` | 普通索引 | 按状态筛选工单列表 |
| `work_orders` | `reporterId` | 普通索引 | 查询某用户的报修记录 |
| `work_orders` | `workerId` | 普通索引 | 查询某维修人员的任务列表 |
| `work_orders` | `categoryId` | 普通索引 | 按分类统计工单 |
| `attachments` | `workOrderId` | 普通索引 | 查询工单附件 |
| `logs` | `workOrderId` | 普通索引 | 查询工单操作日志 |
| `logs` | `userId` | 普通索引 | 查询某用户的操作记录 |
| `notifications` | `(userId, isRead)` | 复合索引 | 查询未读通知 |
| `notifications` | `(userId, createdAt)` | 复合索引 | 分页查询通知列表 |

## 5. 完整的 Prisma Schema

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ========== 枚举 ==========

enum UserRole {
  ADMIN
  WORKER
  STUDENT
  TEACHER
}

enum Priority {
  LOW
  MEDIUM
  HIGH
  URGENT
}

enum WorkOrderStatus {
  PENDING
  ACCEPTED
  ASSIGNED
  PROCESSING
  PENDING_CONFIRM
  COMPLETED
  UNRESOLVED
  CANCELLED
}

enum AttachmentType {
  BEFORE
  AFTER
}

enum NotificationType {
  WORK_ORDER_CREATED
  WORK_ORDER_ACCEPTED
  WORK_ORDER_ASSIGNED
  WORK_ORDER_STARTED
  WORK_ORDER_RESOLVED
  WORK_ORDER_CONFIRMED
  WORK_ORDER_UNRESOLVED
  WORK_ORDER_CANCELLED
}

// ========== 模型 ==========

model User {
  id        Int       @id @default(autoincrement())
  username  String    @unique
  password  String
  name      String
  role      UserRole
  phone     String?
  avatar    String?
  isActive  Boolean   @default(true)
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  reportedOrders  WorkOrder[]    @relation("Reporter")
  assignedOrders  WorkOrder[]    @relation("Worker")
  logs            Log[]
  notifications   Notification[]
}

model Category {
  id          Int         @id @default(autoincrement())
  name        String      @unique
  description String?
  icon        String?
  sortOrder   Int         @default(0)
  isActive    Boolean     @default(true)
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  workOrders  WorkOrder[]
}

model WorkOrder {
  id           Int              @id @default(autoincrement())
  title        String
  description  String
  location     String
  priority     Priority         @default(MEDIUM)
  status       WorkOrderStatus  @default(PENDING)
  resolution   String?
  feedback     String?
  categoryId   Int
  reporterId   Int
  workerId     Int?
  acceptedAt   DateTime?
  assignedAt   DateTime?
  startedAt    DateTime?
  resolvedAt   DateTime?
  confirmedAt  DateTime?
  cancelledAt  DateTime?
  createdAt    DateTime         @default(now())
  updatedAt    DateTime         @updatedAt

  category      Category        @relation(fields: [categoryId], references: [id])
  reporter      User            @relation("Reporter", fields: [reporterId], references: [id])
  worker        User?           @relation("Worker", fields: [workerId], references: [id])
  attachments   Attachment[]
  logs          Log[]
  notifications Notification[]

  @@index([status])
  @@index([reporterId])
  @@index([workerId])
  @@index([categoryId])
}

model Attachment {
  id          Int            @id @default(autoincrement())
  workOrderId Int
  url         String
  filename    String
  mimeType    String
  size        Int
  type        AttachmentType
  createdAt   DateTime       @default(now())

  workOrder   WorkOrder      @relation(fields: [workOrderId], references: [id], onDelete: Cascade)

  @@index([workOrderId])
}

model Log {
  id          Int       @id @default(autoincrement())
  workOrderId Int
  userId      Int
  action      String
  fromStatus  String?
  toStatus    String?
  detail      String?
  createdAt   DateTime  @default(now())

  workOrder   WorkOrder @relation(fields: [workOrderId], references: [id], onDelete: Cascade)
  user        User      @relation(fields: [userId], references: [id])

  @@index([workOrderId])
  @@index([userId])
}

model Notification {
  id          Int              @id @default(autoincrement())
  userId      Int
  workOrderId Int?
  title       String
  content     String
  type        NotificationType
  isRead      Boolean          @default(false)
  createdAt   DateTime         @default(now())

  user        User             @relation(fields: [userId], references: [id])
  workOrder   WorkOrder?       @relation(fields: [workOrderId], references: [id], onDelete: Cascade)

  @@index([userId, isRead])
  @@index([userId, createdAt])
}
```

### 6.1 默认用户

```typescript
// prisma/seed.ts
import { PrismaClient } from '@prisma/client';
import * as bcrypt from 'bcrypt';

const prisma = new PrismaClient();

async function main() {
  // 创建4个默认用户
  const users = [
    {
      username: 'admin',
      password: await bcrypt.hash('Admin123456', 10),
      name: '系统管理员',
      role: 'ADMIN',
      phone: '13800001111',
    },
    {
      username: 'repairer',
      password: await bcrypt.hash('Repair123456', 10),
      name: '李师傅',
      role: 'WORKER',
      phone: '13800002222',
    },
    {
      username: 'student',
      password: await bcrypt.hash('Student123456', 10),
      name: '张三',
      role: 'STUDENT',
      phone: '13800003333',
    },
    {
      username: 'teacher',
      password: await bcrypt.hash('Teacher123456', 10),
      name: '王老师',
      role: 'TEACHER',
      phone: '13800004444',
    },
  ];

  for (const user of users) {
    await prisma.user.upsert({
      where: { username: user.username },
      update: {},
      create: user,
    });
  }

  // 创建4个默认报修分类
  const categories = [
    { name: '水电维修', description: '水龙头、下水道、电路故障等', icon: 'ThunderboltOutlined', sortOrder: 1 },
    { name: '网络故障', description: '校园网、路由器、交换机问题', icon: 'WifiOutlined', sortOrder: 2 },
    { name: '门窗家具', description: '门窗损坏、桌椅维修、锁具更换', icon: 'ToolOutlined', sortOrder: 3 },
    { name: '空调设备', description: '空调不制冷、制热故障、漏水', icon: 'HeatMapOutlined', sortOrder: 4 },
  ];

  for (const category of categories) {
    await prisma.category.upsert({
      where: { name: category.name },
      update: {},
      create: category,
    });
  }

  console.log('Seed data created successfully');
}

main()
  .catch((e) => {
    console.error(e);
    process.exit(1);
  })
  .finally(async () => {
    await prisma.$disconnect();
  });
```
