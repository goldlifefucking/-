## 1. 环境要求

### 1.1 本地开发环境

| 工具 | 最低版本 | 说明 |

| pnpm | 8.x | 包管理工具，支持 workspace |

| Git | 2.x | 版本控制 |

| 依赖 | 说明 |
|------|------|
| Docker + Docker Compose | 生产环境部署 |

| 端口 443 | HTTPS（可选） |
| 端口 3000 | API 服务（仅内部使用，不对外暴露） |
| 端口 5432 | PostgreSQL（仅内部使用，不对外暴露） |

## 2. 本地开发

### 2.1 省流版

```bash
# 1. 克隆项目
git clone <项目仓库地址>
cd campus-work-order

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 修改密码配置（或保持默认）

# 3. 启动 PostgreSQL
docker compose up -d postgres

# 4. 安装依赖
pnpm install

# 5. 初始化数据库
pnpm prisma:generate
pnpm prisma:push
pnpm prisma:seed

# 6. 启动开发服务
pnpm dev
```

### 2.2 详细步骤

#### 步骤 1: 配置环境变量

```bash
cp .env.example .env
```

**.env 文件内容:**

```env
# 数据库连接
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/campus_work_order

# JWT 密钥
JWT_SECRET=your-super-secret-jwt-key-change-in-production

# 上传目录
UPLOAD_DIR=./uploads

# 前端API地址 (Vite 环境变量)
VITE_API_BASE_URL=http://localhost:3000/api
```

> 本地开发使用默认值即可。生产环境请修改 `JWT_SECRET` 为复杂随机字符串。

#### 步骤 2: 启动 PostgreSQL

```bash
docker compose up -d postgres
```

此命令会启动 `docker-compose.yml` 中定义的 PostgreSQL 容器：
- 端口：5432
- 数据库名：`campus_work_order`
- 用户名：`postgres`
- 密码：`postgres`

#### 步骤 3: 安装依赖

```bash
pnpm install
```

此命令会在整个 monorepo 中安装所有依赖：
- `apps/web/` (Vue3 + Vite)
- `apps/api/` (NestJS)
- `packages/shared/` (共享包)

#### 步骤 4: 生成 Prisma Client

```bash
pnpm prisma:generate
```

> 此命令在 `apps/api` 目录下执行 `prisma generate`，根据 `schema.prisma` 生成类型安全的数据库客户端。

#### 步骤 5: 同步数据库表结构

```bash
pnpm prisma:push
```

> 此命令将 Prisma Schema 中的模型同步到 PostgreSQL 数据库，创建所有表。

#### 步骤 6: 初始化种子数据

```bash
pnpm prisma:seed
```

> 此命令运行 `apps/api/src/prisma/seed.ts`，创建默认用户和分类数据。

#### 步骤 7: 启动开发服务

```bash
pnpm dev
```

此命令会同时启动前端和后端开发服务器。

### 2.3 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端页面 | http://localhost:5173 | Vue3 + Vite 开发服务器 |

| Swagger 文档 | http://localhost:3000/api/docs | 交互式 API 文档 |

## 3. Docker 部署（生产环境）

### 3.1 快速部署

```bash
# 1. 克隆项目并进入目录
git clone <项目仓库地址>
cd campus-work-order

# 2. 配置生产环境变量
cp .env.example .env
# 编辑 .env，修改 DB_PASSWORD 和 JWT_SECRET 为安全值

# 3. 构建并启动所有服务
docker compose -f docker-compose.prod.yml up -d --build

# 4. 等待数据库就绪后，初始化种子数据
docker compose -f docker-compose.prod.yml exec api npx prisma db push
docker compose -f docker-compose.prod.yml exec api npx ts-node src/prisma/seed.ts

# 5. 验证服务
curl http://localhost/api/health
```

### 3.2 服务组成

| 容器 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| `postgres` | postgres:16-alpine | 5432 (内部) | 数据库 |
| `api` | 自构建 (NestJS) | 3000 (内部) | 后端API |
| `web` | 自构建 (Nginx + dist) | 80 | 前端 + 反向代理 |

### 3.3 部署流程详解

#### 3.3.1 配置环境变量

生产环境的 `.env` 文件示例：

```env
# 数据库密码（必须修改为强密码）
DB_PASSWORD=A1b2C3d4E5f6G7h8I9j0

# 数据库连接（注意 host 为容器名 postgres）
DATABASE_URL=postgresql://postgres:A1b2C3d4E5f6G7h8I9j0@postgres:5432/campus_work_order

# JWT 密钥（必须修改为复杂随机字符串，建议64字符以上）
JWT_SECRET=K8m2P9vX4Q7wY1N6aB3cD5eF8gH0jL3mN5oP2qR4sT7uV9wX1yZ6ABCdefghIJKL

# 上传文件存储目录
UPLOAD_DIR=/app/uploads
```

#### 3.3.2 构建并启动

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

构建过程：
1. 构建 `api` 镜像：安装依赖 → 编译 TypeScript → 生成 Prisma Client
2. 构建 `web` 镜像：安装依赖 → Vite 构建 → 复制到 Nginx
3. 按依赖顺序启动：postgres → api → web

#### 3.3.3 初始化种子数据

```bash

docker compose -f docker-compose.prod.yml exec api npx prisma db push

# 再运行种子脚本
docker compose -f docker-compose.prod.yml exec api npx ts-node src/prisma/seed.ts
```

## 4. 服务器域名配置

### 4.1 修改 Nginx server_name

编辑 `nginx.conf`，将 `server_name` 改为实际域名：

```nginx
server {
    listen 80;
    server_name work-order.example.com;  # 改为实际域名
    # ... 其余配置保持不变
}
```

修改后重新构建 web 镜像：

```bash
docker compose -f docker-compose.prod.yml up -d --build web
```

### 4.2 配置 DNS 解析

在域名服务商的控制台添加 A 记录：

| 类型 | 主机记录 | 记录值 | TTL |
|------|----------|--------|-----|
| A | work-order | 服务器公网IP | 600 |

### 4.3 配置 HTTPS（可选）

使用 Let's Encrypt 免费证书 + Certbot：

**方式一：独立 Certbot 容器（推荐）**

在 `docker-compose.prod.yml` 中添加 certbot 服务，或使用 `nginx-proxy` + `acme-companion` 组合。

**方式二：手动配置**

```bash
# 1. 安装 certbot
sudo apt update && sudo apt install certbot -y

# 2. 获取证书（standalone 模式，需先停止占用80端口的服务）
docker compose -f docker-compose.prod.yml stop web
sudo certbot certonly --standalone -d work-order.example.com
docker compose -f docker-compose.prod.yml start web

# 3. 修改 nginx.conf 添加 SSL 配置
```

**HTTPS Nginx 配置示例:**

```nginx
server {
    listen 443 ssl;
    server_name work-order.example.com;

    ssl_certificate     /etc/letsencrypt/live/work-order.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/work-order.example.com/privkey.pem;

    client_max_body_size 10m;

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://api:3000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }

    location /socket.io/ {
        proxy_pass http://api:3000/socket.io/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

# HTTP 强制跳转 HTTPS
server {
    listen 80;
    server_name work-order.example.com;
    return 301 https://$server_name$request_uri;
}
```

## 5. 环境变量说明

### 5.1 完整列表

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `DATABASE_URL` | 是 | - | PostgreSQL 连接字符串 |
| `JWT_SECRET` | 是 | - | JWT 签名密钥（生产环境必须修改） |
| `UPLOAD_DIR` | 否 | `./uploads` | 文件上传存储目录 |
| `DB_PASSWORD` | 是 | - | 数据库密码（用于 docker-compose） |
| `VITE_API_BASE_URL` | 否 | `http://localhost:3000/api` | 前端 API 基础地址 |
| `NODE_ENV` | 否 | `development` | 运行环境：development/production |

### 5.2 生产环境强制要求

- `JWT_SECRET`：修改为至少64字符的随机字符串
- `DB_PASSWORD`：修改为强密码（至少16位，含大小写字母+数字+特殊字符）
- `DATABASE_URL`：确保 host 为 Docker 容器名 `postgres`
- 不要将 `.env` 文件提交到 Git 仓库

### 5.3 生成随机密钥

```bash
# 生成 JWT 密钥 (64字符)
openssl rand -base64 48

# 生成数据库密码 (16字符)
openssl rand -base64 12
```

## 6. 常见问题排查

### 6.1 端口被占用

**现象**: `Error: listen EADDRINUSE: address already in use :::3000` 或 `:::80`

**解决**:

```bash
# 查看端口占用
# Windows
netstat -ano | findstr :3000
netstat -ano | findstr :80

# Linux/Mac
lsof -i :3000
lsof -i :80

# 终止占用进程
# Windows (以PID 1234为例)
taskkill /PID 1234 /F

# Linux/Mac
kill -9 <PID>
```

### 6.2 数据库连接失败

**现象**: `Can't reach database server at 'localhost:5432'`

**排查:**

```bash
# 1. 检查 PostgreSQL 容器是否运行
docker compose ps

# 2. 查看数据库日志
docker compose logs postgres

# 3. 确保 PostgreSQL 容器已完全启动 (healthy)
docker compose ps postgres
# 状态应为 "healthy" 而非 "starting"

# 4. 测试连接
docker compose exec postgres psql -U postgres -d campus_work_order
```

### 6.3 Prisma 相关错误

**`prisma:generate` 失败:**

```bash
# 确认在 apps/api 目录下有 schema.prisma
ls apps/api/prisma/schema.prisma

# 重新生成
cd apps/api
npx prisma generate
```

**`prisma:push` 失败:**

```bash
# 确认 DATABASE_URL 正确
echo $DATABASE_URL

# 确认 PostgreSQL 可连接
cd apps/api
npx prisma db push --force-reset  # 注意：这会清空数据！
```

**表已存在但数据丢失:**

```bash
# 重新运行种子脚本
cd apps/api
npx ts-node src/prisma/seed.ts
```

### 6.4 权限问题

**403 Forbidden 错误:**

1. 确认已登录且 Token 有效
2. 确认当前用户角色是否有该操作权限
3. 检查 Token 是否过期（默认7天）
4. 检查请求头：`Authorization: Bearer <token>`

**401 Unauthorized 错误:**

1. Token 未携带或格式错误
2. Token 已过期，需重新登录
3. JWT_SECRET 与签发时不一致

### 6.5 文件上传问题

**上传失败或无权限写入:**

```bash
# 确认 uploads 目录存在且有写入权限
docker compose exec api ls -la /app/uploads

# 如需重建权限
docker compose exec api chmod 755 /app/uploads
```

**前端上传大小限制:**

Nginx 默认限制 1MB，已在 `nginx.conf` 中配置 `client_max_body_size 10m`。

### 6.6 Docker 镜像构建慢

```bash
# 清理旧的 Docker 构建缓存
docker builder prune -a

# 重新构建（不使用缓存）
docker compose -f docker-compose.prod.yml build --no-cache
```

### 6.7 WebSocket 连接失败

**现象**: 实时通知不生效，Socket.IO 连接报错

**排查:**

1. 确认 Nginx 配置中包含 `/socket.io/` 的 WebSocket 代理
2. 确认 `proxy_http_version 1.1` 和 `Upgrade` 头配置正确
3. 如果使用 HTTPS，确认客户端连接地址使用 `wss://` 协议

### 6.8 查看服务日志

```bash
# 查看所有服务日志
docker compose -f docker-compose.prod.yml logs -f

# 查看特定服务日志
docker compose -f docker-compose.prod.yml logs -f api
docker compose -f docker-compose.prod.yml logs -f web
docker compose -f docker-compose.prod.yml logs -f postgres

# 查看最近100行日志
docker compose -f docker-compose.prod.yml logs --tail=100 api
```

### 6.9 数据备份

```bash
# 备份 PostgreSQL 数据
docker compose -f docker-compose.prod.yml exec postgres pg_dump -U postgres campus_work_order > backup_$(date +%Y%m%d).sql

# 恢复数据
docker compose -f docker-compose.prod.yml exec -T postgres psql -U postgres campus_work_order < backup_20250101.sql
```

### 6.10 完全重置

```bash
# 停止并删除所有容器和卷
docker compose -f docker-compose.prod.yml down -v

# 重新部署
docker compose -f docker-compose.prod.yml up -d --build
docker compose -f docker-compose.prod.yml exec api npx prisma db push
docker compose -f docker-compose.prod.yml exec api npx ts-node src/prisma/seed.ts
```
